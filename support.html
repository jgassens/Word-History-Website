Support

Setup and troubleshooting for Word History.

Start with the shared AutoRecover folder setup, then confirm that monitoring is enabled and Word can write recovery files.

Recommended Setup

  1. Create or choose a user-managed AutoRecover folder, such as ~/Documents/WordAutorecovery.
  2. Open Microsoft Word settings and set Word's AutoRecover location to that folder.
  3. Open Word History and choose the same folder as the monitored recovery source.
  4. Keep Word History monitoring enabled while you work in Word.

If Snapshots Are Not Appearing

  • Confirm Word and Word History are pointed at the same AutoRecover folder.
  • Confirm monitoring is enabled in Word History.
  • Make sure Microsoft Word has saved or AutoRecovered the document at least once.
  • Check whether macOS is asking Word History for folder access and approve the prompt if it appears.
  • Restart Word History if the monitoring status appears stale.

If Recovery Does Not Open in Word

  • Confirm Microsoft Word is installed and can open normal .docx files.
  • Try restoring the snapshot as a copy first, then open the copy manually in Word.
  • Check whether Word is showing a modal dialog that needs to be dismissed.

Contact

For support, send a concise description of the problem, your macOS version, Microsoft Word version, Word History version, and the steps that led to the issue.

Contact: open a GitHub issue with the details above.

Do not attach private documents, full recovery filenames, or raw logs to public issues.
