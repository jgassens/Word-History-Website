About

PubMate bridges draft placeholders and EndNote records.

PubMate scans a Word .docx for clearly labeled PMID and DOI markers, fetches matching records, and writes the files EndNote needs to convert temporary citations into formatted references.

  • Reads Explicit PMID markers, DOI labels, DOI resolver URLs, and Word comments when enabled.
  • Writes A separate .endnote.docx, canonical .endnote-import.enw, optional .references.nbib, and JSON report.
  • Protects The input document is not modified; PubMate writes a new output document.
  • Boundary It is for Clarivate EndNote temporary citations, not Microsoft Word endnotes or hand-written CWYW fields.

Screen

Identifiers in. EndNote handoff out.

PubMate keeps the manuscript workflow explicit: labeled identifiers become import records and temporary citations that EndNote can resolve.

How

Temporary citations. Real records.

  1. Use explicit markers such as PMID: 12345678 or DOI: 10.xxxx/xxxxx in the manuscript draft.
  2. Run PubMate on the Word .docx; provide an email address for NCBI requests when prompted.
  3. Import the generated .endnote-import.enw file into the target EndNote library.
  4. Open the generated .endnote.docx in Word and run EndNote's update command.

Support

Support starts with the report file.

PubMate writes a JSON report so failures can be diagnosed without sharing a whole manuscript. Start there, then reduce the document to the smallest redacted example that still reproduces the issue.

  • Include PubMate version, macOS version, EndNote version, marker examples, and relevant report warnings or errors.
  • Check That identifiers are labeled, the reference section was skipped intentionally, and the .enw file was imported into the open EndNote library.
  • Private Do not share an unpublished manuscript; use a redacted document with the smallest failing marker set.
